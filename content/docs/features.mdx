# Features

CelestialDocs is a documentation starter template built with Astro and Shadcn. This page summarizes the key features and how to configure or extend them.

## Overview

- Fast static documentation via Astro content collections
- MDX support for using React/TSX components inside docs
- Configurable site options in `data/config.ts`, grouped into the system metadata (`SITE`), header toggles (`HEADER_FEATURES`), navigation helpers (`HEADER_NAV_ITEMS`, `HEADER_SOCIAL_LINKS`), the sidebar tree (`SIDEBAR_NAVIGATION`), and the right-hand Table of Contents settings (`TABLE_OF_CONTENTS`).
- Auto-generated sidebar and hierarchical navigation
- Search support using Pagefind
- Theme & layout toggles (dark mode, width toggle)
- GitHub integration and optional repo button
- Pluggable UI components (shadcn-based components included)

## Configuration

Edit `data/config.ts` to control site-wide settings. Example:

- `SITE` now holds the global metadata, while `HEADER_FEATURES` controls toggleable header behaviors.
- Use `HEADER_NAV_ITEMS` to drive the header links and `HEADER_SOCIAL_LINKS` for the social buttons.
- Adjust `SIDEBAR_NAVIGATION` when you want to change the sidebar groups or tabs.
- Flip `TABLE_OF_CONTENTS.enableExtra` to toggle the optional markdown-driven ToC extras that appear on the right.

- You can now include *top-level entries* in the sidebar configuration by placing an `Entry` object directly in the `navigation.groups` array. This lets you include single links inline between groups without creating a group for them. Example:

```ts
// navigation.groups array
{
  id: "guides",
  label: "Guides",
  // ...
},
{ slug: "overview/quick-tour", label: "Quick Tour", icon: "sparkles" },
{
  id: "components",
  label: "Components",
  // ...
},
```

This will render the `Quick Tour` link inline between the `Guides` and `Components` entries in the sidebar.

```ts
// data/config.ts
export const SITE = {
  website: "https://example.com",
  author: "Your Name",
  title: "CelestialDocs",
  description: "Documentation site",
  defaultDocRedirect: "/",
};

export const HEADER_FEATURES = {
  enableLayoutWidthToggle: true,
  enableGitHubButton: true,
  starCountThreshold: 5,
};

export const HEADER_NAV_ITEMS = [
  { href: "/blog", label: "Blog" },
];

export const HEADER_SOCIAL_LINKS = [
  { name: "github", href: "https://github.com/you", linkTitle: "GitHub", active: true },
];

export const TABLE_OF_CONTENTS = {
  enableExtra: true,
};
```

### Change the default docs redirect

Set `SITE.defaultDocRedirect` to change where `/docs` redirects when an index doc is missing. Example:

```ts
defaultDocRedirect: "/",
```

## Content structure

Docs live in the `content/docs/` collection. File structure maps to routes, e.g. `content/docs/getting-started/introduction.md` → `/docs/getting-started/introduction`.

Frontmatter fields commonly used:

- `title` — page title
- `description` — short summary
- `navLabel` / `navIcon` — optional navigation label and icon
- `pubDatetime` — publish date (optional)

## Components & Layout

- `src/layouts/DocsLayout.astro` provides the page shell and sidebar
- Reusable components live under `src/components/` (docs-specific components under `src/components/docs/`)
- Many UI components are shadcn-inspired and available in `src/components/ui/`

## Search

Pagefind assets are checked in under `public/pagefind` and `/public/_pagefind`. Search is wired into the site and indexes collection content.

## Theming & Styling

- Tailwind CSS is used site-wide
- Toggle components for dark mode and layout width are available in the header
- Follow the project's Tailwind and component guidelines when creating styles

## Extending & Contributing

- Add a new page under `content/docs/` — it will appear in the docs site automatically
- To add a sidebar group or adjust generated navigation, edit `data/config.ts`'s `SIDEBAR_NAVIGATION` object
- Prefer `mdx` when you need interactive components, otherwise use plain `md` for static content
- Flip `TABLE_OF_CONTENTS.enableExtra` in `data/config.ts` if you want to show or hide the extra right-hand panel items

## Examples

- Add a top-level nav item in `data/config.ts`'s `HEADER_NAV_ITEMS`:

```ts
export const HEADER_NAV_ITEMS = [
  { href: "/blog", label: "Blog" },
];
```

- Create a doc `content/docs/features.mdx` and add frontmatter with `navLabel: "Features"` to expose it in the nav
