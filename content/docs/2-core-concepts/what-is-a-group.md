## Group Structure

A group contains:

```typescript
{
  id: string;                 // Unique identifier for this group
  label: string;              // Display name in sidebar
  icon?: string;              // Emoji or icon name
  tab?: boolean;              // Make this a top-level tab (optional)
  autoGenerated?: boolean;    // Auto-include files from a folder (optional)
  entries?: Entry[];          // Child entries
  groups?: Group[];           // Child groups (nesting)
  path?: string;              // Folder path for auto-generation (optional)
}
```

## Required vs Optional

### Required Fields

**`id`** - Unique identifier
```typescript
{
  id: 'getting-started',  // Must be unique within the sidebar
  label: 'Getting Started'
}
```

**`label`** - Display name
```typescript
{
  id: 'guides',
  label: 'Guides'  // What users see in the sidebar
}
```

### Optional Fields

**`icon`** - Visual indicator
```typescript
{
  id: 'getting-started',
  label: 'Getting Started',
  icon: 'ğŸš€'  // Emoji
}

{
  id: 'reference',
  label: 'Reference',
  icon: 'book'  // SVG from src/assets/icons/book.svg
}
```

**`entries`** - Child entries
```typescript
{
  id: 'getting-started',
  label: 'Getting Started',
  entries: [
    { slug: 'getting-started/introduction' },
    { slug: 'getting-started/installation' },
  ]
}
```

**`groups`** - Nested groups
```typescript
{
  id: 'documentation',
  label: 'Documentation',
  groups: [
    {
      id: 'basics',
      label: 'Basics',
      entries: [
        { slug: 'docs/basics/markdown' }
      ]
    }
  ]
}
```

## Group Behavior

### Expansion/Collapse
- Groups are **collapsible** by default
- Users can click the group label to expand/collapse
- The state is remembered in the browser

### Ordering
- Groups appear in the order you define them
- Inside each group:
  - Manual entries appear first (in your order)
  - Auto-generated entries appear after (alphabetically)

### Icons
- Group icons appear next to the group label
- The same as entry icons: emoji or SVG
