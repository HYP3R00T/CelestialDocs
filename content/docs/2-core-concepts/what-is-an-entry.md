## How Entries Are Created

Entries can be created two ways:

### 1. Automatic Discovery (Filesystem)

If a group has `autoGenerated: true`, all markdown files in that folder become entries:

```sh
content/docs/

   â”œâ”€ installation.md  â† Becomes an entry
```

### 2. Manual Configuration

In `data/config.ts`, explicitly define entries:

```typescript
{
  id: 'getting-started',
  label: 'Getting Started',
  entries: [
    { slug: 'getting-started/introduction' },
    { slug: 'getting-started/installation' },
    { slug: 'getting-started/quick-start' },
  ]
}
```

## Entry Configuration

An entry has these properties:

```typescript
{
  slug: string;           // File path (no .md extension)
  label?: string;         // Sidebar label (optional, uses frontmatter title if omitted)
  icon?: string;          // Emoji or icon name (optional)
  hidden?: boolean;       // Hide from sidebar (optional, default: false)
}
```

### `slug` (Required)

The file path relative to `content/docs/`, **without** the `.md` or `.mdx` extension.

```typescript
// Example slugs:
{ slug: 'getting-started/introduction' }      // content/docs/getting-started/introduction.md
{ slug: 'guides/advanced-config' }            // content/docs/guides/advanced-config.md
{ slug: 'setup' }                             // content/docs/setup.md
```

### `label` (Optional)

Override the sidebar label. If not provided, uses the `title` from the page's frontmatter.

```typescript
{
  slug: 'getting-started/introduction',
  label: 'Intro'  // Sidebar shows "Intro" instead of the page title
}
```

### `icon` (Optional)

Show an emoji or icon next to the entry. Works the same as `navIcon` in frontmatter.

```typescript
{
  slug: 'getting-started/introduction',
  icon: 'ğŸ“š'  // Emoji
}

{
  slug: 'guides/installation',
  icon: 'download'  // SVG icon: src/assets/icons/download.svg
}
```

### `hidden` (Optional)

Hide the entry from the sidebar while keeping it accessible via URL.

```typescript
{
  slug: 'internal/draft-page',
  hidden: true  // Not shown in sidebar, but /docs/internal/draft-page still works
}
```

## Example: Mix of Auto-Generated and Manual Entries

```typescript
{
  id: 'guides',
  label: 'Guides',
  autoGenerated: true,  // Auto-include all files from content/docs/guides/
  entries: [
    // These appear first, in this order
    { slug: 'guides/getting-started', label: 'ğŸš€ Get Started', icon: 'rocket' },
    { slug: 'guides/installation' },
    // Then auto-generated entries follow alphabetically
  ]
}
```

When `autoGenerated: true`:
- Entries you define appear first, in your specified order
- Remaining files are appended alphabetically
- Entries with `hidden: true` are skipped entirely

## When to Use Entries vs Groups

| Use Entries | Use Groups |
|------------|------------|
| Simple navigation structure | Organize related pages |
| Few pages in a category | Many pages needing subcategories |
| Linear documentation | Complex hierarchical docs |
| Product guides | API reference with sections |
