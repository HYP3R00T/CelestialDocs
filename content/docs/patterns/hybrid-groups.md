Hybrid groups are one of CelestialDocs' most powerful navigation features. They let you combine manually specified entries with auto-generated discovery, giving you precise control where you need it while maintaining the convenience of automation.

## What are Hybrid Groups?

A hybrid group is a navigation group that:

1. **Starts with manual entries** - Pages you explicitly list in a specific order
2. **Followed by auto-generated entries** - Remaining pages discovered automatically

This gives you the best of both worlds: control over important pages and automatic discovery of the rest.

## Why Use Hybrid Groups?

Hybrid groups are perfect when you want to:

- **Pin important pages** - Keep key pages at the top
- **Maintain order** - Control the sequence of critical content
- **Scale automatically** - New pages appear without configuration updates
- **Reduce maintenance** - Don't manually list every single page

## Controlling Auto-Generated Order

Auto-generated entries are sorted alphabetically by default. You can control their order using the `order` field in frontmatter.

### Using Order Field

```markdown
---
title: "Important Feature"
description: "A critical feature"
order: 1  # Lower numbers appear first
---

# Important Feature
```

```markdown
---
title: "Another Feature"
description: "Another feature"
order: 2  # Appears after order: 1
---

# Another Feature
```

### Order Strategy

```typescript
{
  id: "features",
  label: "Features",
  icon: "âœ¨",
  entries: [
    { slug: "features/overview" },  // No order needed (manual)
  ],
  autoGenerated: true,
}
```

With frontmatter:

```markdown
<!-- features/core-features.md -->
---
title: "Core Features"
order: 1
---

<!-- features/advanced-features.md -->
---
title: "Advanced Features"
order: 2
---

<!-- features/experimental.md -->
---
title: "Experimental Features"
order: 3
---
```

Result:

1. Overview (manual)
2. Core Features (order: 1)
3. Advanced Features (order: 2)
4. Experimental Features (order: 3)

## Excluding Files from Auto-Generation

Use the `navHidden` frontmatter field to exclude files from navigation:

```markdown
---
title: "Internal Documentation"
description: "For internal use only"
navHidden: true  # Won't appear in navigation
---

# Internal Documentation
```

This is useful for:

- Draft pages
- Internal documentation
- Deprecated pages
- Template files

## Troubleshooting

### Manual Entry Not Appearing

- Check that the file exists at the specified slug
- Verify the slug path is correct (no leading slash, no extension)
- Ensure the file has valid frontmatter with a title

### Auto-Generated Entries Not Showing

- Verify `autoGenerated: true` is set
- Check that files exist in the correct directory
- Ensure files have valid frontmatter
- Check that `navHidden` is not set to `true`

### Wrong Order

- Manual entries always appear first, in the order listed
- Auto-generated entries are sorted alphabetically by default
- Use the `order` field in frontmatter to control auto-generated order
- Check file names for alphabetical sorting

### Duplicate Entries

If a file is listed manually AND matches auto-generation:

- The manual entry takes precedence
- The file won't appear twice
- Remove from manual entries if you want it auto-generated

## Comparison: Manual vs Auto vs Hybrid

### Manual Groups

```typescript
{
  id: "guides",
  label: "Guides",
  icon: "ğŸ“–",
  entries: [
    { slug: "guides/page1" },
    { slug: "guides/page2" },
    { slug: "guides/page3" },
  ],
  // No autoGenerated
}
```

**Pros:** Complete control over order
**Cons:** Must update config for every new page

### Auto-Generated Groups

```typescript
{
  id: "guides",
  label: "Guides",
  icon: "ğŸ“–",
  autoGenerated: true,
  // No manual entries
}
```

**Pros:** Zero maintenance, scales automatically
**Cons:** No control over order (except via frontmatter)

### Hybrid Groups

```typescript
{
  id: "guides",
  label: "Guides",
  icon: "ğŸ“–",
  entries: [
    { slug: "guides/overview" },
  ],
  autoGenerated: true,
}
```

**Pros:** Control + automation, best of both worlds
**Cons:** Slightly more complex to understand

## When to Use Hybrid Groups

Use hybrid groups when:

âœ… You have a few critical pages that should always be first
âœ… You want new pages to appear automatically
âœ… You have a growing section that needs to scale
âœ… You want to reduce configuration maintenance

Don't use hybrid groups when:

âŒ You need complete control over every entry's position
âŒ You have only 2-3 pages total (just use manual)
âŒ The order doesn't matter at all (just use auto-generated)
