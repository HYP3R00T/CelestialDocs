This page is a developer-focused reference for the `data/config.ts` file. It explains each top-level option, what it controls, and gives concrete examples for common changes and extensions.

> Note: the runtime types for these options are defined in `src/lib/types.ts`. If you make changes here, keep the TypeScript types in sync.

---

## SITE (site metadata)

The `SITE` object contains global metadata used throughout the site (branding, links, SEO defaults, and the default `/docs` redirect).

Important fields:
- `website` — the canonical site URL.
- `author` — author or organization name.
- `repo` — repository URL displayed by the GitHub button and useful for linking.
- `branch` — optional default branch name used in links (if needed).
- `title` — site title shown in the header and meta tags.
- `description` — site description used in meta tags.
- `image` / `imageAlt` — global preview image and alt text (commonly imported from `src/assets` like `import setupImage from "@/assets/setup.png"`).
- `twitterHandle` — used to populate social meta tags.
- `CONTENT.systems` — an array of content systems; each system may include a `defaultDocRedirect` that controls where the collection root redirects (e.g., a system object `{ id: 'docs', defaultDocRedirect: '/docs/getting-started/introduction' }`).

Example:

```ts
export const SITE = {
  website: 'https://example.com',
  author: 'Acme Inc',
  repo: 'https://github.com/acme/docs',
  title: 'Acme Docs',
  description: 'Official docs for Acme products',
  image: setupImage,
  imageAlt: 'Acme Docs preview',
};
```

Example (CONTENT):

```ts
export const CONTENT = {
  dir: 'content/docs',
  defaultDocRedirect: '/docs/getting-started/introduction',
};
```

Changing `CONTENT.defaultDocRedirect` updates where the top-level "Docs" header link sends users.

## LOCALE

Simple locale configuration for the site. At minimum set `lang` to the site's language code:

```ts
export const LOCALE = { lang: 'en' };
```

## HEADER_FEATURES, HEADER_NAV_ITEMS, HEADER_SOCIAL_LINKS

These objects control header UI and navigation behavior:

- `HEADER_FEATURES` — toggles and thresholds for header features:
  - `starCountThreshold`: minimum star count to display a star indicator (if integrated).
  - `enableLayoutWidthToggle`: show/hide the layout width toggle.
  - `enableGitHubButton`: show/hide the GitHub button.

- `HEADER_NAV_ITEMS` — an array of `NavItem` objects used to populate header links. Fields: `{ href, label, special?, blank? }`. Example:

```ts
export const HEADER_NAV_ITEMS = [
  { href: '/docs', label: 'Docs', special: true },
  { href: 'https://example.com/blog', label: 'Blog', blank: true },
];
```

- `HEADER_SOCIAL_LINKS` — array of `SocialObjects`: `{ name, href, active, linkTitle? }`. Use `active: true` to show a link.

```ts
export const HEADER_SOCIAL_LINKS = [
  { name: 'github', href: 'https://github.com/acme', linkTitle: 'Acme on GitHub', active: true },
];
```

Notes:
- `blank: true` opens the link in a new tab.
- Keep labels short for header visibility on small screens.

## SIDEBAR_NAVIGATION (core docs navigation)

`SIDEBAR_NAVIGATION` defines top-level groups and manual entries for the docs sidebar. The type is `Sidebar` (`src/lib/docs/types.ts`). For a full user-level guide see the _Sidebar Configuration_ docs.

Important fields and behaviors:
- `groups` — an array of `Group` or `Entry` objects. A `Group` may include: `id`, `label`, `icon`, `path`, `tab`, `autoGenerated`, `groups`, `entries`.
- `autoGenerated` — when `true`, pages under the group's path are appended automatically.
- `tab` — mark a group as a tab (used by `buildTabs` to generate top-level tabs)
- `entries` — manual entries (`{ slug, label?, icon?, hidden? }`) pinned at the top of the group.

Example:

```ts
export const SIDEBAR_NAVIGATION = {
  groups: [
    {
      id: 'getting-started',
      label: 'Getting Started',
      icon: 'rocket',
      autoGenerated: true,
      entries: [ { slug: 'getting-started/introduction' } ],
    },
  ],
};
```

Practical tips:
- Use `autoGenerated: true` for low maintenance; add `entries` to pin key pages.
- Use `path` to gather pages from a directory that doesn't match `id`.
- Use `tab: true` on groups you want presented as tabs in the sidebar UI.

## TABLE_OF_CONTENTS

Controls the right-side on-page TOC behavior. Current options:

- `enableExtra` (boolean) — show an extra panel under the TOC for additional content (`src/components/docs/TableOfContentsExtra.astro`).

Example:

```ts
export const TABLE_OF_CONTENTS = { enableExtra: false };
```

---

## Icons and assets

- Sidebar and entry icons: use emoji in frontmatter (easiest) or put SVGs in `src/assets/icons/` and reference by name (e.g., `navIcon: 'pattern'`). `DocIcon.astro` looks for the SVG and falls back to a default document icon.
- Site images referenced in `SITE.image` are commonly imported from `src/assets/` so bundlers can optimize them (e.g., `import setupImage from '@/assets/setup.png'`).

---

## Developer notes & tips for changes

- Many changes in `data/config.ts` are read at server render time; while `pnpm dev` will usually pick them up instantly, double-check by restarting the dev server if you don't see changes.
- The navigation builder logs warnings when manual slugs are missing — use this to find misconfigured entries.
- Keep config edits small and test them locally before merging (e.g., change a group label and verify in the dev server).
