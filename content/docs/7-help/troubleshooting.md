## Pages not showing in sidebar

**Problem:** You created a page but it doesn't appear in the sidebar.

**Solutions:**

1. **Check `autoGenerated` setting:**
   - If `autoGenerated: false`, manually add the entry to `SIDEBAR_NAVIGATION`
2. **Check for `navHidden: true`:**
   - If `navHidden: true`, the page won't appear (but URL works)
3. **Check file path:**
   - Make sure the slug matches your file path exactly
   - File: `content/docs/guides/my-page.md`
   - Slug: `guides/my-page`
4. **Check frontmatter:**
   - Page must have `title` and `description`

## Sidebar shows wrong order

**Problem:** Pages appear in random order, not as configured.

**Solutions:**

1. **Use numeric prefixes:**
   ```
   1-getting-started.md
   2-installation.md
   3-configuration.md
   ```
2. **Use manual entries in config:**
   ```typescript
   entries: [
     { slug: 'getting-started' },
     { slug: 'installation' },
     { slug: 'configuration' }
   ]
   ```
3. **Avoid auto-generation** if you need exact control

## Theme toggle not working

**Problem:** Dark/light mode toggle is missing or broken.

**Solutions:**

1. **Check if enabled** in `HEADER_FEATURES`
2. **Hard refresh browser** (Ctrl+Shift+R or Cmd+Shift+R)
3. **Clear browser cache** and try again

## Dev server not detecting changes

**Problem:** You saved a file but changes don't appear.

**Solutions:**

1. **Restart dev server:**
   ```bash
   npm run dev
   ```
2. **Clear build cache:**
   ```bash
   rm -rf .astro
   npm run dev
   ```
3. **Hard refresh browser:**
   - Ctrl+Shift+R (Windows)
   - Cmd+Shift+R (Mac)

## Sidebar configuration not updating

**Problem:** You changed `SIDEBAR_NAVIGATION` but sidebar doesn't update.

**Solutions:**

1. **Save the file** - Make sure `data/config.ts` is saved
2. **Restart dev server:**
   ```bash
   npm run dev
   ```
3. **Check for TypeScript errors** - Syntax mistakes prevent loading
4. **Clear cache:**
   ```bash
   rm -rf .astro
   npm run dev
   ```

## Build fails

**Problem:** Running `npm run build` shows errors.

**Solutions:**

1. **Check console output** - Look for specific error messages
2. **Verify file paths** - All slugs in config must match real files
3. **Check TypeScript** - Run `npm run astro check`
4. **Look for drafts** - Files with `draft: true` might cause issues

## Common Error Messages

### "Collection not found"

- **Cause:** Slug in config doesn't match any file
- **Fix:** Check file path matches slug exactly

### "Undefined reference"

- **Cause:** Icon or image file not found
- **Fix:** Verify file exists and path is correct

### "Type error in config"

- **Cause:** Wrong type in configuration
- **Fix:** Check `data/config.ts` syntax and types
