Use the `SIDEBAR_NAVIGATION` object in `data/config.ts` to define how documentation pages are grouped and ordered in the sidebar.

## Group fields

- `id` (string) â€” unique group identifier (required). Typically matches the directory under `content/docs/` (e.g., `getting-started`).
- `label` (string) â€” the display name for the group in the sidebar.
- `path` (string) â€” optional path override. If set, the group will collect pages from this path instead of `id`.
- `icon` (string) â€” optional icon name (or emoji) shown next to the group label.
- `autoGenerated` (boolean) â€” when `true` the builder will include filesystem pages from the group's path that are not explicitly configured as entries. When `false`, only the configured `entries` will be shown.
- `entries` (array) â€” an array of manual entries to display for the group. Each entry is `{ slug, label?, icon?, hidden? }`.
- `groups` (array) â€” nested groups to create multi-level navigation.

## Entry fields

- `slug` (string) â€” the page slug (e.g., `getting-started/introduction`). This should match the file path without extension.
- `label` (string) â€” optional override for the entry label shown in the sidebar.
- `icon` (string) â€” optional icon name or emoji for this entry.
- `hidden` (boolean) â€” if `true`, the manual entry will not be shown in the sidebar.

## Examples

### 1) Basic auto-generated group

This will create a "Getting Started" group and append any pages under `content/docs/getting-started` automatically, while highlighting two manual entries at the top:

```ts
export const SIDEBAR_NAVIGATION = {
  groups: [
    {
      id: 'getting-started',
      label: 'Getting Started',
      icon: 'rocket',
      autoGenerated: true,
      entries: [
        { slug: 'getting-started/introduction' },
        { slug: 'getting-started/installation' },
      ],
    },
  ],
};
```

### 2) Manual-only group (turn off auto-generation)

Use this when you want full control over entries and order:

```ts
{
  id: 'tutorials',
  label: 'Tutorials',
  autoGenerated: false,
  entries: [
    { slug: 'tutorials/getting-started', label: 'Start here' },
    { slug: 'tutorials/advanced', label: 'Advanced topics' },
  ],
}
```

### 3) Nested groups + path override

You can define nested groups and use `path` when the folder name does not match the desired group id:

```ts
{
  id: 'guides',
  label: 'Guides',
  groups: [
    {
      id: 'advanced',
      label: 'Advanced',
      path: 'guides/advanced',
      autoGenerated: true,
    },
  ],
}
```

### 4) Hiding pages and using `navHidden`

- To hide a page from the sidebar while keeping it accessible by URL, set `navHidden: true` in the page frontmatter. This is useful for partial pages or internal notes.
- To hide breadcrumbs on a per-page basis, set `hide_breadcrumbs: true` in a page's frontmatter.
- To hide a manual entry configured in `data/config.ts`, set `hidden: true` on that entry.


## Practical tips & best practices

- Prefer `autoGenerated: true` for most content areas â€” it's low maintenance and picks up new pages automatically.
- Use manual `entries` when you want to pin important pages to the top, provide a different label, or create a curated order.
- Use `path` to gather pages from a different directory structure without renaming the group id.
- When adding a manual slug, make sure the target page exists â€” the build will warn if a configured entry's slug is not found.
- Use emojis for quick, readable icons (`navIcon: 'ðŸ“š'`) or add a custom SVG to `src/assets/icons/` and reference its name (e.g., `navIcon: 'pattern'`).

## Troubleshooting

- If an expected page does not appear in the sidebar: check the page's frontmatter for `navHidden: true`, verify the slug path, and ensure the page file exists under `content/docs/`.
- If you see a console warning like `Top-level entry <slug> not found`, it means a manual entry refers to a non-existent page â€” either add the page or remove the entry.

## When to edit `data/config.ts` vs page frontmatter

- Edit page-level controls (labels, icons, hidden flags) in the page frontmatter when the page author controls presentation.
- Edit `data/config.ts` to structure the overall sidebar (group labels, manual entries, top-level ordering, and special grouping behavior).
