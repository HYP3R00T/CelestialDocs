## Navigation Levels Explained

### Level 1: Tabs

Tabs are the highest level of organization. They create distinct navigation contexts in your sidebar:

```typescript
{
  id: "patterns",
  label: "Common Patterns",
  icon: "ğŸ¯",
  tab: true,  // This makes it a tab
  autoGenerated: true,
}
```

When a user clicks a tab, the entire sidebar switches to show that tab's content. This is perfect for separating major sections like:

- Main Documentation vs API Reference
- User Guides vs Developer Docs
- Tutorials vs Reference Material

### Level 2: Groups

Groups organize related pages within a tab or the default tab:

```typescript
{
  id: "getting-started",
  label: "Getting Started",
  icon: "ğŸš€",
  entries: [
    { slug: "getting-started/installation" },
    { slug: "getting-started/quick-start" },
  ],
}
```

Groups appear as collapsible sections in the sidebar, making it easy to scan and navigate related content.

### Level 3: Entries

Entries are individual documentation pages. They're the leaves of your navigation tree:

```typescript
{ slug: "getting-started/installation" }
```

Each entry corresponds to one Markdown or MDX file in your content directory.

## Best Practices for Nested Navigation

### Keep It Shallow

While CelestialDocs supports three levels of nesting, that doesn't mean you should always use all three. Aim for the shallowest structure that works:

**Good:**

```

â””â”€â”€ Getting Started
    â”œâ”€â”€ Installation
    â””â”€â”€ Quick Start
```

**Avoid:**

```

    â””â”€â”€ Subcategory
        â””â”€â”€ Sub-subcategory
            â””â”€â”€ Page
```

### Use Tabs Sparingly

Tabs are powerful but can hide content. Use them for truly distinct sections:

âœ… **Good use of tabs:**

- User Documentation (tab) vs Developer API (tab)
- Tutorials (tab) vs Reference (tab)
- Main Docs (default) vs Examples (tab)

âŒ **Poor use of tabs:**

- Installation (tab) vs Configuration (tab) - these should be groups
- Version 1 (tab) vs Version 2 (tab) - use versioning instead

## Hybrid Groups

You can also create hybrid groups that combine manual entries with auto-generation:

```typescript

  id: "guides",
  label: "Guides",
  icon: "ğŸ“–",
  entries: [
    { slug: "guides/overview" },  // Always first
    // Auto-generated entries will appear here
  ],
  autoGenerated: true,  // Also discover other files
}
```

This is perfect when you want one or two pages at the top, followed by alphabetically sorted pages.

## Navigation Depth Limits

CelestialDocs enforces a maximum depth of 3 levels to maintain usability:

- **Level 1:** Tab (or Default Tab)
- **Level 2:** Group
- **Level 3:** Entry

If you find yourself needing more depth, consider:

1. **Splitting into multiple tabs** - Separate major sections
2. **Using page sections** - Use headings and table of contents within pages
3. **Reorganizing content** - Sometimes less nesting is better
4. **Creating landing pages** - Use overview pages to link to related content

## Troubleshooting Nested Navigation

### Navigation Not Appearing

- Check that your group IDs match your folder names
- Verify that files exist at the specified slugs
- Make sure `autoGenerated: true` is set if you want auto-discovery
- Restart your dev server after configuration changes

### Wrong Order

- For manual groups, entries appear in the order you list them
- For auto-generated groups, entries are sorted alphabetically
- Use the `order` field in frontmatter to control auto-generated order

### Tabs Not Working

- Ensure `tab: true` is set on the group
- Check that the tab has content (entries or autoGenerated)
- Verify there are no duplicate tab IDs
