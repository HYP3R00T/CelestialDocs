The sidebar is the primary navigation structure for your documentation. CelestialDocs provides a powerful and flexible sidebar configuration system that supports manual entries, auto-generation, tabs, groups, and mixed approaches.

## Overview

Sidebar navigation is configured in the `SIDEBAR_NAVIGATION` object in `data/config.ts`. Each content system has its own independent sidebar configuration.

```typescript
export const SIDEBAR_NAVIGATION: SidebarNavigation = {
    docs: {
        defaultTab: {
            label: "Documentation",
            icon: "document",
        },
        groups: [
            // Navigation groups go here
        ],
    },
    funnydocs: {
        defaultTab: {
            label: "FunnyDocs",
            icon: "ğŸ¤¡",
        },
        groups: [
            // Navigation groups go here
        ],
    },
};
```

## Core Concepts

### Navigation Hierarchy

The sidebar uses a three-level hierarchy:

1. **Tabs** - Top-level navigation contexts (optional)
2. **Groups** - Categorical containers for related entries
3. **Entries** - Individual documentation pages

### Default Tab

Every content system has a default tab that contains groups not explicitly marked as tabs:

```typescript
defaultTab: {
    label: "Documentation",  // Display name
    icon: "document",        // SVG icon reference
}
```

### Groups

Groups organize related documentation pages. They can be:

- **Manual** - Explicitly list entries in a specific order
- **Auto-generated** - Automatically discover files from a folder
- **Hybrid** - Combine manual entries with auto-generation
- **Tab-level** - Act as a separate tab in the navigation

## Configuration Structure

### System Configuration

Each content system in `SIDEBAR_NAVIGATION` has this structure:

```typescript
{
    defaultTab: {
        label: string,    // Display name for the default tab
        icon: string,     // Icon (emoji or SVG reference)
    },
    groups: Array<{
        id: string,              // Unique identifier
        label: string,           // Display name
        icon?: string,           // Optional icon
        tab?: boolean,           // Is this a tab-level group?
        autoGenerated?: boolean, // Auto-discover files?
        entries?: Array<{        // Manual entries (optional)
            slug: string,        // File path without extension
            label?: string,      // Override display label
            icon?: string,       // Override icon
        }>,
    }>,
}
```

## Auto-Generated Groups

Auto-generated groups automatically discover and include all files from a folder. Files are sorted alphabetically by their title.

### Basic Auto-Generated Group

```typescript
{
    id: "features",
    label: "Features",
    icon: "âœ¨",
    autoGenerated: true,
}
```

This will automatically include all files in `content/docs/features/` and sort them alphabetically.

### When to Use Auto-Generation

**Use auto-generation when:**

- âœ… Alphabetical order makes sense
- âœ… You have many files that change frequently
- âœ… Order doesn't matter for understanding
- âœ… You want to reduce configuration maintenance

**Use manual configuration when:**

- âœ… Specific order is important (e.g., tutorials)
- âœ… You want to exclude certain files
- âœ… You need custom labels or icons
- âœ… You want fine-grained control

## Hybrid Groups

Hybrid groups combine manual entries with auto-generation. Manual entries appear first, followed by auto-generated entries.

```typescript
{
    id: "advanced",
    label: "Advanced",
    icon: "ğŸ”¬",
    autoGenerated: true,
    entries: [
        // These appear first, in this order
        { slug: "advanced/overview" },
        { slug: "advanced/getting-started" },
        // Then all other files in advanced/ folder (alphabetically)
    ],
}
```

## Field Reference

### Group Fields

#### id

- **Type:** `string`
- **Required:** Yes
- **Description:** Unique identifier for the group

Must match the folder name in your content directory:

```typescript
id: "getting-started"  // Matches content/docs/getting-started/
id: "core-concepts"    // Matches content/docs/core-concepts/
```

#### label

- **Type:** `string`
- **Required:** Yes
- **Description:** Display name shown in the sidebar

```typescript
label: "Getting Started"
label: "Core Concepts"
label: "API Reference"
```

#### icon

- **Type:** `string`
- **Required:** No
- **Description:** Icon displayed next to the group label

Can be an emoji or SVG reference:

```typescript
icon: "ğŸš€"           // Emoji
icon: "document"     // SVG reference (from src/assets/icons/)
```

#### tab

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- **Description:** Whether this group is a tab-level group

```typescript
tab: true   // This group appears as a separate tab
tab: false  // This group appears in the default tab
```

#### autoGenerated

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- **Description:** Whether to automatically discover files

```typescript
autoGenerated: true   // Auto-discover files from folder
autoGenerated: false  // Use manual entries only
```

#### entries

- **Type:** `Array<Entry>`
- **Required:** No (required if `autoGenerated` is false)
- **Description:** Array of manual entries

```typescript
entries: [
    { slug: "getting-started/index" },
    { slug: "getting-started/installation" },
]
```

### Entry Fields

#### slug

- **Type:** `string`
- **Required:** Yes
- **Description:** File path relative to content directory, without extension

```typescript
slug: "getting-started/index"           // content/docs/getting-started/index.md
slug: "core-concepts/navigation-system" // content/docs/core-concepts/navigation-system.md
```

#### label

- **Type:** `string`
- **Required:** No
- **Description:** Override the display label (uses frontmatter title by default)

```typescript
{
    slug: "getting-started/index",
    label: "Introduction"  // Override default title
}
```

#### icon

- **Type:** `string`
- **Required:** No
- **Description:** Override the icon (uses frontmatter navIcon by default)

```typescript
{
    slug: "getting-started/installation",
    icon: "âš¡"  // Override default icon
}
```

## Best Practices

### Group Organization

**Keep groups focused:**

- Each group should have a clear theme
- Limit to 5-10 entries per group
- Use tabs for distinctly different content

**Good Example:**

```typescript
{
    id: "getting-started",
    label: "Getting Started",
    entries: [
        { slug: "getting-started/introduction" },
        { slug: "getting-started/installation" },
        { slug: "getting-started/quick-start" },
    ],
}
```

**Avoid:**

```typescript
{
    id: "everything",
    label: "Everything",
    entries: [
        // 50+ entries - too many!
    ],
}
```

### Naming Conventions

**IDs:**

- Use kebab-case
- Match folder names exactly
- Keep them descriptive

```typescript
id: "getting-started"  // Good
id: "core-concepts"    // Good
id: "Getting Started"  // Bad - use kebab-case
id: "gs"              // Bad - not descriptive
```

**Labels:**

- Use title case
- Keep them concise
- Make them descriptive

```typescript
label: "Getting Started"           // Good
label: "Core Concepts"             // Good
label: "getting-started"           // Bad - use title case
label: "Everything You Need to Know About Getting Started"  // Bad - too long
```

### Icon Usage

**Choose appropriate icons:**

- Use emoji for friendly, approachable sections
- Use SVG icons for technical sections
- Be consistent within each group

```typescript
// Friendly sections - use emoji
{ id: "getting-started", label: "Getting Started", icon: "ğŸš€" }
{ id: "guides", label: "Guides", icon: "ğŸ“š" }

// Technical sections - use SVG or emoji
{ id: "api", label: "API Reference", icon: "code" }  // SVG
{ id: "configuration", label: "Configuration", icon: "âš™ï¸" }  // Emoji
```

### Entry Order

**For manual groups:**

- Order by learning progression (beginner to advanced)
- Put essential content first
- Group related topics together

**For auto-generated groups:**

- Ensure file names sort logically
- Use number prefixes if order matters: `01-intro.md`, `02-setup.md`
- Or use frontmatter `navLabel` to control display names

### Tab Usage

**Use tabs for:**

- Distinctly different content types (Guides vs API Reference)
- Different audiences (Users vs Developers)
- Supplementary content (Patterns, Examples, Help)

**Don't overuse tabs:**

- Limit to 2-4 tabs maximum
- Most content should be in the default tab
- Too many tabs can confuse users

## Troubleshooting

### Pages Not Appearing in Sidebar

**Problem:** Files exist but don't show in navigation

**Solutions:**

1. **For manual groups:** Verify the slug matches the file path exactly
2. **For auto-generated groups:** Ensure the group ID matches the folder name
3. **Check frontmatter:** Verify files have `title` and `description`
4. **Check navHidden:** Ensure `navHidden` is not set to `true` in frontmatter

```typescript
// File: content/docs/getting-started/installation.md
// Correct slug:
{ slug: "getting-started/installation" }

// Wrong slugs:
{ slug: "installation" }                    // Missing folder
{ slug: "getting-started/installation.md" } // Don't include extension
{ slug: "/getting-started/installation" }   // Don't start with /
```

### Wrong Order in Auto-Generated Groups

**Problem:** Files appear in unexpected order

**Solution:** Auto-generated groups sort alphabetically by title. To control order:

1. **Use number prefixes in filenames:**

   ```
   01-introduction.md
   02-installation.md
   03-quick-start.md
   ```

2. **Use navLabel in frontmatter:**

   ```yaml
   ---
   title: "Installation Guide"
   navLabel: "01. Installation"
   ---
   ```

3. **Switch to manual configuration** if you need precise control

### Icons Not Displaying

**Problem:** Icons don't appear or show as broken

**Solutions:**

1. **For emoji:** Ensure you're using a valid Unicode emoji
2. **For SVG:** Verify the icon file exists in `src/assets/icons/`
3. **Check the reference:** SVG references should not include `.svg` extension

```typescript
icon: "ğŸš€"           // Good - emoji
icon: "document"     // Good - SVG reference
icon: "document.svg" // Bad - don't include extension
icon: "missing"      // Bad - file doesn't exist
```

### Tabs Not Appearing

**Problem:** Tab-level groups don't create separate tabs

**Solutions:**

1. Verify `tab: true` is set
2. Ensure the group has content (entries or auto-generated files)
3. Check that the folder exists and contains files
4. Clear browser cache and rebuild

### Group ID Mismatch

**Problem:** Auto-generation doesn't work

**Solution:** The group ID must exactly match the folder name:

```typescript
// Folder: content/docs/getting-started/
// Correct:
{ id: "getting-started", autoGenerated: true }

// Wrong:
{ id: "gettingStarted", autoGenerated: true }  // Wrong case
{ id: "getting_started", autoGenerated: true } // Wrong separator
{ id: "started", autoGenerated: true }         // Wrong name
```
