The **sidebar** is the left panel on documentation pages. It shows:

- Groups of related pages
- Individual page links (entries)
- Tabs for organizing by content type
- Icons and emojis for visual organization

## What is the Sidebar?

The sidebar helps readers:
- Understand the structure of your documentation
- Find related pages quickly
- Navigate hierarchically
- See what content is available

## How the Sidebar is Configured

In `data/config.ts`, the `SIDEBAR_NAVIGATION` object controls everything:

```typescript
export const SIDEBAR_NAVIGATION = {
  docs: {  // For the docs collection
    defaultTab: { label: "Docs", icon: "book" },
    groups: [
      // Your groups and entries here
    ]
  },
  notes: {  // For the notes collection
    defaultTab: { label: "Notes", icon: "file-text" },
    groups: [
      // Your groups and entries here
    ]
  }
}
```

Each collection (docs, notes, etc.) has its own sidebar configuration.

## Sidebar Building Blocks

The sidebar is built from three concepts:

| Concept | Purpose | Example |
|---------|---------|---------|
| **Entry** | Links to a single page | "Installation", "Quick Start" |
| **Group** | Container for entries and subgroups | "Getting Started", "Reference" |
| **Tab** | Top-level grouping with its own icon | "Docs", "Guides", "Reference" |

## How Sidebar Works

1. **Discovery** - All `.md` files in `content/docs/` are found
2. **Grouping** - Files are grouped by folder structure
3. **Configuration** - `SIDEBAR_NAVIGATION` controls order, labels, icons
4. **Rendering** - The sidebar is built and displayed to users

## Automatic vs Manual

### Automatic (Filesystem-driven)
If you set `autoGenerated: true`:
```typescript
{
  id: 'guides',
  label: 'Guides',
  autoGenerated: true  // All content/docs/guides/* files become entries
}
```

### Manual (Configured)
You explicitly list pages:
```typescript
{
  id: 'getting-started',
  label: 'Getting Started',
  entries: [
    { slug: 'getting-started/introduction' },
    { slug: 'getting-started/installation' }
  ]
}
```

### Mixed
Combine both approaches:
```typescript
{
  id: 'guides',
  label: 'Guides',
  autoGenerated: true,
  entries: [
    { slug: 'guides/featured-guide' }  // This appears first
    // Then auto-generated entries follow alphabetically
  ]
}
```

## Configuration Structure

```typescript
export const SIDEBAR_NAVIGATION = {
  docs: {
    // Optional: label and icon for the default/first tab
    defaultTab: {
      label: "Docs",
      icon: "book"
    },

    // Array of groups (and/or tabs)
    groups: [
      {
        id: "group-id",
        label: "Group Label",
        icon: "icon-name",  // Optional
        tab: true,          // Optional: makes this a top-level tab
        entries: [ ... ],   // Optional: individual page links
        groups: [ ... ]     // Optional: nested groups
      }
    ]
  }
}
```

## Key Concepts

### defaultTab
Optional. Appears as the first tab when the sidebar opens.

### tab: true
Makes a group appear as a clickable tab at the sidebar top.

### autoGenerated: true
Automatically discovers and includes all files in a folder.

### entries
Array of individual page links (Entry objects).

### groups
Array of nested groups (Group objects).
