Control **how entries appear** in the sidebar and **whether they appear at all**.

## Controlling Entry Order

### Option 1: Manual Configuration Order

In `SIDEBAR_NAVIGATION`, entries appear in the order you list them:

```typescript
entries: [
  { slug: 'guides/getting-started' },      // 1st
  { slug: 'guides/installation' },         // 2nd
  { slug: 'guides/configuration' },        // 3rd
  { slug: 'guides/advanced' }              // 4th
]
```

**Sidebar shows (top to bottom):**
```sh
Getting Started
Installation
Configuration
Advanced
```

### Option 2: Numeric Prefixes in Filenames

Use numbers to control order:

```sh
content/docs/guides/
â”œâ”€ 1-getting-started.md
â”œâ”€ 2-installation.md
â”œâ”€ 3-configuration.md
â””â”€ 4-advanced.md
```

**Sidebar shows (in order):**
```sh

Configuration
Advanced
```

### Option 3: navLabel with Prefixes

Use `navLabel` in frontmatter:

```yaml
---
title: "Getting Started"
navLabel: "01 Getting Started"
---
```

The `01` doesn't show but controls sorting.

## Mixed Manual and Auto-Generated Order

When you have both manual and auto-generated entries:

**Configuration:**
```typescript
{
  id: 'guides',
  label: 'Guides',
  autoGenerated: true,
  entries: [
    { slug: 'guides/getting-started' },  // Manual: appears first
    { slug: 'guides/installation' }      // Manual: appears second
    // Auto-generated follow
  ]
}
```

**Result:**
```sh
Getting Started      (manual, order 1)
Installation         (manual, order 2)
Advanced            (auto, alphabetical)
Configuration       (auto, alphabetical)
FAQ                 (auto, alphabetical)
Troubleshooting     (auto, alphabetical)
```

## Hiding Pages from Sidebar

### Method 1: Frontmatter `navHidden`

Set `navHidden: true` in a page's frontmatter:

```yaml
---
title: "Draft Page"
navHidden: true
---
```

**Result:**
- âŒ Page does NOT appear in sidebar
- âœ… Page IS still accessible via URL: `/docs/draft-page`
- âœ… Page can be linked to directly

**Use cases:**
- Draft pages in progress
- Utility pages
- Internal documentation
- Archived content

### Method 2: Configuration `hidden` Property

In `SIDEBAR_NAVIGATION`, set `hidden: true`:

```typescript
entries: [
  { slug: 'guides/getting-started' },
  { slug: 'guides/draft', hidden: true }  // Hidden from sidebar
]
```

**Result:**
- âŒ Page does NOT appear in sidebar
- âœ… Page IS still accessible via URL
- âœ… Can be visited if someone has the link

## Filtering Out Drafts

Drafts are automatically skipped by:
- Auto-generation (`autoGenerated: true`)
- Hidden entries (`navHidden: true`)

You **don't need to do anything** - CelestialDocs handles it automatically.

## Tips & Best Practices

- âœ… **Use numeric prefixes** - `1-`, `2-`, `3-` for automatic ordering
- âœ… **Use `navHidden` for drafts** - Keep them in version control but hidden
- âœ… **Pin important pages first** - Use manual entries for featured content
- âœ… **Keep entries logical** - Follow the user's learning path
- âœ… **Hide utility pages** - Privacy policy, legal, meta pages
- âŒ **Don't rely on alphabetical order** - Use prefixes instead
- âŒ **Don't have mixed numeric and non-numeric files** - Choose one approach
- âŒ **Don't hide too many pages** - Users should find what they need
